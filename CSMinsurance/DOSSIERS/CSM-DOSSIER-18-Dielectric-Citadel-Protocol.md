## SECTION 2: CORE DELIVERABLE — THE DIELECTRIC CITADEL PROTOCOL

### 1. PROTOCOL OVERVIEW

#### 1.1. Definition
The **Dielectric Citadel Protocol (DCP)** is an integrated framework for achieving electromagnetic infrastructure independence at critical nodes within interdependent infrastructure networks. It combines physical hardening (materials, devices, system architecture), operational resilience (planning, training, procedures), and financial mechanisms (insurance incentives, capital allocation) to ensure that critical infrastructure survives a Carrington-class CME with minimal loss of function.

#### 1.2. Core Principles
1. **Node protection, not blanket coverage**: Identify and harden the critical nodes whose failure triggers cascading infrastructure collapse. Not every electronic device needs protection — only the ones whose failure is systemically consequential.
2. **Layered defense**: No single hardening measure is sufficient. Combine physical shielding (AEGIS-C), current blocking (NBDs), operational response (GIC monitoring), and recovery capability (strategic spares).
3. **Insurance as the economic driver**: Use insurance pricing — premium credits for hardened assets, higher premiums for unhardened — to create a market incentive for hardening. The insurance industry funds the Citadel through the same mechanism it uses to manage every other risk: accurate pricing.
4. **Certification and standardization**: Establish clear, verifiable hardening standards (DCP Tiers 0-3) that risk engineers can assess, underwriters can price, and policyholders can achieve.
5. **Global applicability**: The Citadel architecture is designed for international deployment. GIC physics and EMI physics are universal. The solutions work in any country, any grid architecture, any infrastructure type.

### 2. CITADEL ARCHITECTURE — LAYERS OF PROTECTION

#### 2.1. Layer 1: Electromagnetic Shielding (AEGIS-C)
- **Function**: Attenuate incident EM fields (GIC, EMP E1/E2/E3, RF) before they reach sensitive electronics.
- **Technology**: AEGIS-C composite shielding system (ZrB₂-SiC + MXene, SE = 148-165 dB, 1 kHz - 10 GHz).
- **Application points**: Transformer control cabinets, substation control rooms, data center server enclosures, generator start controllers, SCADA/PLC enclosures, communications equipment.
- **Standard**: DCP Tier 2 requires SE ≥148 dB on all critical control electronics. Tier 3 requires SE ≥148 dB on all electronics in the protected facility.

#### 2.2. Layer 2: Current Blocking (Neutral Blocking Devices)
- **Function**: Block quasi-DC GIC from entering power transformers through neutral-ground connections.
- **Technology**: Capacitor-based or resistor-based NBDs installed in transformer neutral-ground path. Capacitive NBDs block DC entirely. Resistive NBDs limit GIC magnitude.
- **Application points**: All transformers ≥100 MVA or ≥345 kV in DCP Tier 2 or higher facilities. Transformers ≥50 MVA or ≥230 kV for Tier 1.
- **Standard**: DCP Tier 2 requires NBDs on all at-risk transformers. Tier 1 requires GIC monitoring capability with NBDs planned.

#### 2.3. Layer 3: Operational Resilience (GIC Monitoring and Response)
- **Function**: Provide real-time situational awareness of GIC conditions, enabling preemptive operator actions before transformer damage occurs.
- **Technology**: GIC monitoring (Hall effect or Rogowski coil sensors on transformer neutrals), SWPC alert subscription, documented escalation procedures for G3 → G4 → G5 alerts.
- **Standard**: DCP Tier 1 requires SWPC alert subscription and documented response procedures. Tier 2 adds GIC monitoring hardware. Tier 3 adds automated protection response.

#### 2.4. Layer 4: Recovery Capability (Strategic Spares and Fuel Autonomy)
- **Function**: Ensure rapid recovery if hardening is overcome by an event exceeding design basis.
- **Technology**: Strategic spare transformers (RecX standardized designs, regionally prepositioned), extended generator fuel autonomy (>168 hours / 7 days minimum), multi-supplier fuel contracts with independent pumping capability.
- **Standard**: DCP Tier 2 requires ≥168 hours fuel autonomy and verified refueling capability. Tier 3 adds strategic spare transformer contract or on-site spare unit.

#### 2.5. Layer 5: Communications Independence
- **Function**: Maintain communications capability when cellular, internet, and SATCOM are degraded or unavailable.
- **Technology**: Multi-mode communications: satellite phone (Iridium/Inmarsat with hardened antenna), HF radio (NVIS for regional), hardwired landline (POTS), LoRa mesh network for local device-to-device communications.
- **Standard**: DCP Tier 2 requires at least two independent communications modes. Tier 3 requires three modes plus on-site communications technician.

### 3. CRITICAL NODE IDENTIFICATION — PRIORITIZATION METHODOLOGY

#### 3.1. Node Criticality Score
For each node in the infrastructure network, calculate a Criticality Score:
```
CS = N × F × R × I
```
Where:
- **N (Network Centrality)**: Number of downstream nodes dependent on this node. Range: 1-10.
- **F (Failure Consequence)**: Severity of consequence if node fails (1 = inconvenience, 5 = loss of life possible, 10 = mass casualties probable).
- **R (Recovery Time)**: Time to restore node function after failure (1 = hours, 5 = weeks, 10 = months/years).
- **I (Interdependence)**: Number of other infrastructure sectors dependent on this node (1 = single sector, 5 = 2-3 sectors, 10 = 4+ sectors: power, water, telecom, healthcare, fuel, etc.).
