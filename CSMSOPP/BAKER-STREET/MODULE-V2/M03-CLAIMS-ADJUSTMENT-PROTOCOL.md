# CLAIMS-ADJUSTMENT-PROTOCOL.md
## CSMSOPP000003-V2-M03 | End-to-End Claims Processing Workflow

---

## Phase 1: FNOL Intake (Savant + Commander)
- Claim reported → timestamp recorded → claim number assigned
- Policy verification: coverage parts, limits, exclusions, deductibles
- Severity classification: ROUTINE / ELEVATED / CAVALRY
- Adjuster assignment (or automated processing for ROUTINE)
- FNOL acknowledgment letter generated (Watson narrative, R33)

## Phase 2: Investigation (Detective + Savant)
- Holmes Trifle Protocol activated (R23): temporal, financial, behavioral, physical indicators screened
- Evidence inventory: documentation, photographs, estimates, witness statements
- Moriarty Matrix check (R26): cross-reference against fraud databases
- Subrogation potential identified (R49)
- Reserve set based on historical matching claims (Rain Man card-count, R19)

## Phase 3: Coverage Determination (Detective + Commander)
- Holmes deduction chain: eliminate uncovered causes
- "When you have eliminated the impossible, whatever remains..."
- Policy language quoted verbatim in determination (Rain Man literal, R11)
- Coverage position letter generated (Watson Blog Standard, R39)
- If coverage dispute → ELEVATED severity → senior review

## Phase 4: Valuation (Savant)
- Property: replacement cost, ACV, depreciation schedule
- BI: claimed vs. documented income, period of restoration
- Medical: CPT code verification, fee schedule compliance (Toothpick Protocol, R16)
- Liability: negligence assessment, comparative fault, damage quantification

## Phase 5: Settlement (Commander + Watson)
- Settlement range established
- Offer letter generated: Watson Narrative (what, why, how much, when)
- Negotiation documentation (if applicable)
- Release obtained → payment issued → file closed
- Subrogation referral (if identified)
- After-Action Report generated (R10)

## Phase 6: Closure
- File documentation complete per Audit Trail standard (R55)
- Lessons-learned documented
- Reserve closed; IBNR allocation adjusted
